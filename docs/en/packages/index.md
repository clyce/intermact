# Monorepo package layers

The **complete symbol list, parameters, and return values** for the public API are in **[API Reference](/en/reference/)** — generated by TypeDoc from exports and TSDoc comments in `packages/*/src` (`pnpm run gen:reference`). Do not hand-write duplicate API pages in the docs site.

## Layout

```text
packages/
  core/           @intermact/core
  render-three/   @intermact/render-three
  render-r3f/     @intermact/render-r3f
  react/          @intermact/react
examples/         @intermact/examples (private, demo gallery)
docs/             @intermact/docs (private, this docs site)
```

## Dependency rules

| Package | May depend on | Forbidden |
| --- | --- | --- |
| `core` | itself, pure algorithm libraries | React, three, DOM |
| `render-three` | `core`, three | React, DOM |
| `render-r3f` | `core`, `render-three`, R3F | — |
| `react` | all of the above | — |

`pnpm run depcruise` validates this in CI.

## Maintaining API documentation

1. Write TSDoc (`/** … */`) for **public exports** in source code
2. Edit [`reference-index.en.src.md`](https://github.com/clyce/intermact/blob/main/docs/reference-index.en.src.md) to update the **API Reference overview** architecture notes (Phase-1 & Phase-2 summary; Phase-3 symbol pages are generated by TypeDoc)
3. Run `pnpm run gen:reference` (automatically run by `dev:docs` / `build:docs`)
4. Output goes to `docs/reference/`, mounted by VitePress
